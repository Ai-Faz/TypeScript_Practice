    **What is SPFx?
     =  SPFx (SharePoint Framework) is a modern client-side development model used to build custom web parts and extensions in Microsoft SharePoint.
        It uses technologies like React and TypeScript, and runs entirely in the browser without server-side code.
    
    **Why SPFx?
     =  SPFx is used because it provides a modern, client-side development model for Microsoft SharePoint.
        It allows developers to build responsive and scalable solutions using technologies like React and TypeScript.
        Unlike traditional server-side development, SPFx runs entirely in the browser, which improves performance and makes it cloud-friendly.
        It also integrates securely with SharePoint data using REST APIs and Microsoft Graph.

        or

        SPFx is preferred because it supports a modern development toolchain including Node.js, npm, Yeoman, Gulp, and TypeScript, which makes development faster and more efficient.
        It allows us to easily consume REST APIs to interact with SharePoint data.
        SPFx supports both SharePoint Online and On-Premises environments like SharePoint 2016 and 2019.
        It is purely client-side, which improves performance and eliminates the need for server-side deployment.
        Additionally, it is platform-independent and works across operating systems like Windows and macOS.

        or

        SPFx is different from older SharePoint development models because it does not rely on JavaScript injection and does not use iframes like the add-in model.
        In traditional approaches, JavaScript was injected into pages, which was harder to maintain and less secure.
        Also, add-ins used iframes, which caused performance issues and poor user experience.
        SPFx, on the other hand, runs directly in the page context, providing better performance, seamless integration, and a more modern development experience in Microsoft SharePoint.

    **Main Features of SPFx
     =  The main features of SPFx include flexibility in choosing frameworks like React, Angular, or Knockout for development.
        It supports modern open-source tools such as npm, TypeScript, Yeoman, Webpack, and Gulp, which makes development efficient and scalable.
        SPFx components are responsive by default, so they work well across devices like mobile, tablet, and desktop.
        It can be used in both classic and modern SharePoint pages, although it is mainly optimized for modern experiences.
        Additionally, SPFx follows a secure model and does not support elevation of privileges, ensuring safer development within Microsoft SharePoint.

    **SPFx Toolchain 
     = Node.js
        Used to build and run SPFx applications
        Provides runtime environment for development tools
        SPFx requires specific LTS versions (earlier versions supported Node 8.x, newer SPFx uses higher LTS versions)
     - NPM (Node Package Manager)
        Manages and installs required packages
        Maintains an online repository of reusable libraries
        Packages can be installed:
        Locally → npm install
        Globally → npm install -g
        Installed packages are stored in the node_modules folder
     - Gulp
        Task runner used to automate development tasks
        Handles:
        Build process
        Packaging & deployment
        Bundles and minifies JavaScript & CSS files
        Compiles TypeScript into JavaScript
     -TypeScript
        Primary language used in SPFx development
        Adds strong typing over JavaScript
        Improves code quality and maintainability
        Compiled into JavaScript during build
     -Yeoman
        Used to scaffold (generate) SPFx project structure
        Helps create ready-to-use project templates
        Command: yo @microsoft/sharepoint
     -Webpack
        Module bundler used in SPFx
        Combines multiple files into a single optimized bundle
        Improves performance and loading speed
     -Local SharePoint Workbench
        Runs on local machine using Node.js
        Used for testing SPFx components without deploying to SharePoint
        Available as workbench.html
        Faster for development and debugging
        Does not require SharePoint connection
     -Hosted SharePoint Workbench
        Runs inside Microsoft SharePoint (SharePoint Online / On-Premises)
        Used to test components with real SharePoint data
        Available as workbench.aspx (not “aspy” ❌)
        Requires deployment or hosting in SharePoint environment
        Helps validate real-world behavior
