📌 Project Overview
This project demonstrates how to create a SharePoint List dynamically using an SPFx (SharePoint Framework) Web Part.
User ek input box me list ka naam deta hai aur button click karte hi backend me SharePoint REST API call hota hai aur list create ho jaati hai.

🧠 How It Works (Simple Flow)
User Input → Button Click → Event Trigger → API Call → SharePoint List Created

🏗️ Step-by-Step Explanation

🔹 Step 1: UI Render (render method)
public render(): void {  this.domElement.innerHTML = `...`;}
👉 Ye method Web Part ka UI render karta hai
👉 Isme:


Left side → Info Card


Right side → Input + Button



🧾 Important UI Elements
<input type="text" id="NewSPList" /><input type="button" id="CreateNewSPList" />
👉 Ye 2 cheeze important hai:


Input → list name lene ke liye


Button → action trigger karne ke liye



🔹 Step 2: Event Binding
private attachEvents(): void {  const button = this.domElement.querySelector('#CreateNewSPList');  if (button) {    button.addEventListener('click', () => this.createSPList());  }}
👉 Button click hone par:
➡️ createSPList() function call hota hai

🔹 Step 3: Input Read Karna
const input = this.domElement.querySelector('#NewSPList') as HTMLInputElement;const listName = input.value.trim();
👉 User jo naam dalega wo yaha se milta hai

🔹 Step 4: Validation
if (!input || !input.value.trim()) {  alert("Please enter list name");  return;}
👉 Empty input prevent karne ke liye

🔹 Step 5: API URL
const url = this.context.pageContext.web.absoluteUrl + "/_api/web/lists";
👉 Ye SharePoint REST endpoint hai
👉 Yaha request bhejne se new list create hoti hai

🔹 Step 6: Payload (Data bhejna)
const listDef = {  Title: listName,  BaseTemplate: 100};
📌 Meaning:


Title → List ka naam


BaseTemplate: 100 → Custom List



🔹 Step 7: API Call (IMPORTANT)
this.context.spHttpClient.post(  url,  SPHttpClient.configurations.v1,  { body: JSON.stringify(listDef) })
👉 Ye sabse important part hai
💡 Why use spHttpClient?


Automatically handles:
Authentication
Request Digest
Headers




👉 Isliye manually headers dene ki zarurat nahi

🔹 Step 8: Response Handle Karna
.then(async (res: SPHttpClientResponse) => {  if (res.ok) {    alert("List Created Successfully");  } else {    const text = await res.text();    alert("Error: " + text);  }})
📌 Important:


res.ok → success check karta hai (200–299)
Error case me actual message show hota hai



🔹 Step 9: Button UX Handling
button.disabled = true;button.value = "Creating...";
👉 User experience improve karne ke liye

🔹 Step 10: Finally Block
.finally(() => {  button.disabled = false;  button.value = "Create List";});
👉 Button ko wapas normal state me lana

⚠️ Important Learnings (Very Important)
❌ Avoid:
Manual headers (Accept / Content-Type confusion)
__metadata (modern API me issue deta hai)
__REQUESTDIGEST manually lena

✅ Use:
spHttpClient
Minimal payload
res.ok for success check
